# Milestone: V9 Executable Kernel

## Status

Complete; ready for owner approval and not merged.

## Goal

Turn SWECircuit's checked file-based protocol into a small, portable executable kernel with versioned contracts, deterministic validation, project initialization, read-only trace inspection, and evidence-backed private package consumption.

## Shipped

- Adopted SWECircuit as the repository and project identity while preserving historical provenance and making no package, CLI, domain, or registry claim.
- Added strict v1alpha1 schemas for Project, Module, Circuit, WorkPacket, RunEvent, and AdapterManifest.
- Added deterministic project validation for paths, references, graph composition, permissions, ownership, limits, and stable diagnostics.
- Added offline, collision-safe project initialization with guarded ownership and immediate validation.
- Added bounded read-only inspection of caller-owned JSONL traces for causation, attempts, retries, outcomes, evidence, terminal state, and privacy suppression.
- Added a concise current-versus-target README, reviewed workflow visual, executable source-checkout quick start, and private internal CLI renderer.
- Added a measured dogfood circuit with controlled failures, caller recovery, retry lineage, digest-bound evidence, and identity-checked cleanup.
- Added a clean packed-consumer gate that installs the private tarball offline into an isolated consumer and runs init, validate, and inspect from installed `node_modules`.

## Why It Matters

SWECircuit now has an executable contract layer that IDEs, agents, and CI can call consistently without pretending the repository already schedules agents, executes adapters, merges code, writes traces, or manages memory automatically. Future orchestration can build on tested module, circuit, work-packet, event, permission, and evidence contracts instead of inventing those boundaries per integration.

## Residual Risks

- No license has been selected, so public reuse terms remain unclear.
- The package and CLI are private source-checkout interfaces; no publication or namespace ownership is claimed.
- V9 validates contracts and inspects traces but does not launch agents, schedule work, write traces, execute adapters, merge branches, or update memory automatically.
- Pure Node retains documented create/capture/removal race windows and one opaque Windows reparse-metadata boundary.
- Trace inspection cannot prove producer truth, append-only history, or retention policy.
- External reviewer liveness still requires manual deadlines, central closure, and retry.
- One local dogfood timing observation is not a benchmark or production-performance claim.

## Next Recommended Work

- Design the smallest V10 slice around one external executor adapter or bounded work-packet handoff without moving orchestration authority into the kernel.
- Plan and test the 0.x `rail` path migration to public Circuit terminology without rewriting historical evidence.
- Resolve licensing before inviting broad external reuse or package distribution.

## User-Facing Overview

V9 turns SWECircuit from a checked workflow handbook into an executable contract kernel. It can initialize projects, validate modular AI-development circuits, inspect retryable execution traces, and prove its private artifact works from a clean consumer across Node 22 and 24 on Windows, macOS, and Linux. It remains deliberately honest: agent execution, orchestration, merging, trace writing, and automatic memory are future adapters, not hidden claims.
