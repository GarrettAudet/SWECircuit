# Milestone: V10 Bounded Executor Boundary

## Status

Complete; immutable acceptance candidate `fa4371e` passed exact local verification, all seven hosted jobs, and correctness, security, and API/documentation review. Ready for owner approval and not merged.

## Goal

Add the smallest honest execution layer above V9: one bounded work packet, one host-injected executor, one invocation-scoped authority grant, and one inspectable lifecycle journal, without claiming grant consumption or replay prevention.

## Shipped

- A provider-neutral `executeWorkPacket` library boundary for one host-selected packet.
- Closed grant, manifest, identity, permission, timeout, snapshot, privacy, and settlement gates before or around invocation.
- Deterministic V9-compatible lifecycle journals with honest cancellation, timeout, and `abort_unconfirmed` outcomes.
- A no-I/O deterministic test executor, installed embedding guide, public TypeScript types, and clean consumer proof.
- V10 dogfood evidence, diagnostics `SC4201` through `SC4210`, workflow checker coverage, research, ADR, feature records, and durable memory updates.

## Why It Matters

V10 is the first executable bridge from a validated SWECircuit work packet to a trusted IDE or agent runtime. It standardizes the handoff and evidence shape without pretending that core schedules work, enforces a sandbox, owns credentials, or controls merge.

## Residual Risks

- The embedding host, not SWECircuit, must sandbox executor code and enforce the checked grant.
- A blocking in-process executor can stall timers and cancellation; unacknowledged aborts remain potentially live.
- V10 has no scheduler, retry engine, provider loader, durable writer, merge automation, or automatic memory mutation.
- The 0.x API may change after real adapter use; public reuse remains legally unclear without a license.
- The active-Markdown checker is bounded defense in depth, not a full CommonMark renderer; parser replacement or contract narrowing remains separately scoped.

## Next Recommended Work

- Request the explicit owner decision on merging V10 to `main`.
- After approval, merge the reviewed branch, verify main-branch CI, record merge provenance, and use V10 as the next dogfood baseline.
- After V10 adoption, build the smallest provider-independent orchestration planner that turns a goal and module circuit into dependency-bounded work packets, matches required capabilities to available specialized agents, schedules safe fan-out and fan-in, and emits an integration plan plus trace.
- Keep IDE, model, and provider integrations as replaceable execution adapters; they expose capabilities and execute assigned packets but do not own decomposition or routing policy. Optional policy compilers may implement that SWECircuit-owned policy only by emitting portable framework contracts.
