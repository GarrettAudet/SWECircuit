# Milestone: V5 TraceRail Modular Orchestration Framework

## Status

Ready for approval on branch `codex/v5-modular-orchestration-framework`.

## Goal

Name the system TraceRail and create a lightweight framework layer that can absorb best-in-class tools and practices through modular contracts and optional adapters while keeping the core workflow simple and traceable.

## Shipped

- Added `docs/framework/` with framework overview, module registry, orchestration patterns, module template, adapter evaluation template, decomposition template, and orchestration run template.
- Promoted modular framework guidance into `AGENTS.md`, the handbook, the PR template, and the checker.
- Added a dated V5 research snapshot and practice-register updates.
- Added V5 source artifacts, review, milestone, and memory pointers.

## Why It Matters

V5 turns the workflow from a strong template into TraceRail: a portable framework kernel. It gives future projects a way to plug in Spec Kit, BMAD, LangChain, AutoGen, CrewAI, Codex subagents, memory tools, retrieval tools, or verification tools only when the contract and value are clear.
