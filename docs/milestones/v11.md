## Goal

Give any IDE a built-in, provider-neutral way to turn one clarified software goal into an optimized roster of exact task-shaped specialists, while preserving a serial baseline, hard quality and authority gates, deterministic selection evidence, and a launch package that binds what the user reviewed.

## Why It Matters

Before V11, an IDE could invent generic roles, over-split fixed work, duplicate context and authority, or launch a roster different from the one a user reviewed. The Specialist Compiler makes the team itself auditable: every candidate partitions the same reviewed work, the serial path remains visible, correctness and requested evidence independence are hard gates, and the selected contracts are portable across external hosts.

This establishes the pure planning boundary needed before a later runtime layer without coupling provider choice, execution, isolation, retry, merge, or memory effects into compiled task demand.

## Dogfood Outcome

The [revision-6 dogfood report](../specs/v11-specialist-compiler/evidence/dogfood/report.json) compiled six stable release work units. Exact search evaluated 203 canonical candidates, found 52 eligible, and selected six task-shaped agents at projected makespan 23 with peak concurrency 4. The serial baseline projected 40 and was ineligible because one agent could not satisfy requested producer/reviewer independence.

The selected-versus-serial values are deterministic planning units. They do not establish wall-clock speed, runtime capacity, isolation, or successful provider execution. The external host performed the recorded specialist work and preserved the handoffs; V11 core constructed and verified the contracts.
