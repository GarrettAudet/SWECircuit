## Goal

Give a developer one IDE/API/model/provider-neutral workflow that takes a software goal through user-authored modules and policy, bounded decomposition, specialized capability assignment, safe parallel work, integrated verification/review, owner approval, complete trace, and evidence-backed learning.

No V11 schema, source, package, scheduler, executor, automatic merge, or durable memory behavior has shipped.

## Why It Matters

This is real dogfood of the intended system. One architecture goal was decomposed into four bounded specialist reviews, executed in parallel without write conflicts, tied to immutable commits, and integrated by one owner. Twice, fan-in found material defects and stopped implementation. Revision 3 turns that evidence into machine-checkable policy and contracts before code makes errors expensive.

## Revision-3 Design

- One `runGoal` facade covers start and continuation; requested concurrency defaults to one and must be explicit above one.
- A closed PolicyBundle binds exact Circuit, Module, and WorkPacket-template values; host policy alone defines bounded replication.
- GoalContract owns criterion coverage; the compiler derives producer, verifier, reviewer, integration, and evidence bindings.
- Logical WorkPacket roles remain distinct from runtime profile/executor assignments.
- Root RunAuthority plus content-bound transition/journal cursors bind every handoff.
- Formal matching, conflict-safe waves, claim reserves, attempt replay prevention, and installed claims control parallel work.
- One-shot dispatch observations and four child variants distinguish stopped, settled, rejected, and unknown effects.
- Two-pass batches, queued requests, cancellation, joins, cycles, memory proposals/candidates, terminals, paths, privacy, and bytes have total rules.
- MergeReadyEvidence is produced only after completed state/event closure; core still never merges or mutates durable memory.
- Core still does not choose IDE/API/model/provider, merge, retry automatically, or mutate memory.
