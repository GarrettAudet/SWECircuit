# Milestone: V11 Portable Orchestration Control Plane

## Status

Draft architecture bootstrap. No V11 runtime capability is implemented, accepted, or merge-ready.

## Goal

Let a human goal move through user-defined SWECircuit modules into bounded work packets, capability-matched specialized agents, dependency-safe parallel waves, integration gates, and one source-preserving execution trace without making an IDE, API, model, or provider the workflow authority.

## Shipped

The branch currently contains planning evidence only:

- A V11 feature package with acceptance criteria, work-unit contracts, test obligations, diagnosis placeholders, and review gates.
- A dated primary-source ecosystem scan focused on portable orchestration practices.
- Proposed ADR 0003 defining the control-plane and execution-plane boundary.
- A decomposition plan and orchestration record for independent design review.
- Active-context, architecture-index, and retrieval pointers for the stacked planning state.

No kernel, schema, package, scheduler, executor, automatic merge, or durable memory behavior has shipped.

## Why It Matters

This architecture gate converts the product goal into a testable boundary before code expands. It makes decomposition, specialized assignment, parallel readiness, clarification, fan-in, integrated quality gates, and trace ownership explicit while preserving hosts as replaceable execution environments.

## Residual Risks

- The artifact, event-version, matching, claim, isolation-assertion, and public-API decisions remain open.
- Stateless revision checks cannot provide cross-process exclusivity without atomic host persistence.
- Safe overlapping writes require real host-enforced workspace isolation.
- V11 is stacked on an owner-gated baseline and may require rebase and full revalidation.
- Public reuse remains legally unclear until the owner selects a license.

## User-Facing Overview

V11 is not almost shipped yet; its architecture is defined for independent review but is not technically accepted. The intended capability is clear: give SWECircuit a goal and selected modules, receive a validated decomposition, assign specialized agents by capability, run safe work in parallel through host adapters, integrate and verify every result, and retain a complete trace. This draft milestone exists to show that progress without implying the runtime already works.
