# Milestone: V3 Development Milestones And Branch Flow

## Status

Complete.

## Goal

Define the edit state this repository should move toward: clean committed baselines, one active version branch at a time, and human-readable milestone overviews at version closeout.

## Shipped

- Development milestone system under `docs/milestones/`.
- Milestone template and V1/V2/V3 summaries.
- Desired edit state section in the handbook.
- Branch and merge workflow section in the handbook.
- Agent-facing milestone and branch rules.
- PR template prompts for milestone and branch/merge state.
- Checker requirements for milestone docs and new handbook sections.

## Why It Matters

V3 makes progress visible from start to finish. Specs and reviews preserve evidence; milestones summarize what each version accomplished; branch rules keep future version work isolated until approved and merged to `main`.
