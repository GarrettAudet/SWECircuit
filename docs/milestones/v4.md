# Milestone: V4 Standalone Agent And IDE Baseline

## Status

Merged to `main` as part of the approved baseline.

## Goal

Make the standalone single-agent/IDE workflow strong enough before expanding into multi-agent development, and make the workflow visibly active while the user chats with the IDE agent.

## Shipped

- IDE-user interaction protocol under `docs/ide/`.
- Reusable IDE message templates for start banners, stage updates, clarification, diagnosis, verification, and handoff.
- Standalone agent work-package docs under `docs/agents/`.
- Reusable standalone agent work-package template.
- Handbook sections for `IDE User Interaction Protocol` and `Standalone Agent And IDE Workflow`.
- Agent entrypoint rules requiring workflow visibility and single-agent excellence before multi-agent expansion.
- Checker coverage for IDE docs, agent docs, and handbook sections.
- Memory updates for IDE interaction and single-agent-first decisions, patterns, glossary, and retrieval pointers.

## Why It Matters

V4 makes the workflow visible and usable in the actual IDE conversation. The user can see which stage the agent is in, what artifact is active, what assumptions are being made, what evidence exists, and what gate comes next. It also strengthens the single-agent baseline before any multi-agent expansion.
