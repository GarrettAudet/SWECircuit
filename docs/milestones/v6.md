# Milestone: V6 Module And Rail Catalog

## Status

Ready for approval on branch `codex/v6-module-rail-catalog`.

## Goal

Normalize TraceRail around catalogs of rails, modules, and packs so the framework can stay simple at the core while growing through optional extensions.

## Shipped

- Added `docs/rails/` with feature, diagnosis, decomposition, adapter, and release rails.
- Added `docs/modules/` with core modules and best-practice adapter modules.
- Added `docs/packs/` with optional pack tiers, lifecycle, template, and conformance guidance.
- Updated README, AGENTS, handbook, framework docs, registry, PR template, checker, memory, and V6 source artifacts.
- Hardened `AGENTS.md` as the concise agent contract and routing index with catalog routes and evergreen tool-policy wording.
- Added first official pack, `tracepack-orchestration-readiness`, as a manual dependency-free pilot for safe multi-agent fan-out.
- Added pack conformance checker coverage for official pack contract fields.

## Why It Matters

V6 makes TraceRail feel more like a real composable framework. Best-in-class repos now fit as modules, rails, adapters, or packs instead of loose recommendations.

## User-Facing Overview

V6 adds the catalog layer: rails, modules, and packs. TraceRail now has a simple core primitive plus a clear way to compose workflows, map best-in-class repos into modules, let a community extend the system without bloating core, keep `AGENTS.md` as a clear agent contract, and pilot a first official orchestration-readiness pack without installing runtime tools.
