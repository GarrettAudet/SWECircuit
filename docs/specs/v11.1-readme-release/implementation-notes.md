## Durable Learnings

- A concise public surface needs stronger, not weaker, executable truth guards.
- Review the scaled output, not only the source canvas.
- Natural-language capability guards need bounded subjects and grammar variants, plus truthful-negative controls.
- Public documentation tests are part of the compatibility contract even when the prose is being intentionally shortened.
- Host translations must distinguish an agent's requested effects from read-only tools the host uses to supply declared context.
- Preserve verifier-rejected bytes and corrected successors; do not silently rewrite evidence.
- A runtime input must be delivered as exact bytes, not implied by a path or digest mentioned in the host prompt.
- Source-bound public files preserve truth but can widen release cost; future trust-root tiering must retain exact freshness while narrowing unrelated reapproval.
