# Spec Module

## Purpose

Define what success means before implementation.

## Input

- Clarified goal.
- Requirements.
- Constraints.
- Assumptions.

## Action

Write testable behavior, acceptance criteria, risks, non-goals, and architecture impact.

## Output

- Feature spec.
- Acceptance criteria.
- Risks and constraints.

## Gate

Acceptance criteria are testable, or work routes to `clarify` or `split`.

## Outcome

`pass`, `clarify`, or `split`.

## Artifacts

- `docs/specs/.../spec.md`

## Adapter

Spec Kit is an optional adapter candidate for automating this module.
