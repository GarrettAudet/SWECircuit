# Clarify Module

## Purpose

Resolve ambiguity before planning or implementation.

## Input

- User request.
- Existing context.
- Open questions.

## Action

Ask only questions that change the plan, or record low-risk assumptions.

## Output

- Clarified requirements.
- Recorded assumptions.
- Remaining open questions.

## Gate

Requirements are clear enough to spec, or work routes to `clarify` or `block`.

## Outcome

`pass`, `clarify`, or `block`.

## Artifacts

- Feature spec assumptions.
- Open questions.
- IDE clarification prompt.

## Adapter

Superpowers-style questioning can support this module as an optional capability adapter.
