# Review Module

## Purpose

Compare implementation to intent, architecture, risks, evidence, and memory requirements.

## Input

- Spec.
- Diff.
- Verification evidence.
- Architecture constraints.
- Memory changes.

## Action

Audit alignment, scope, risks, tests, docs, and follow-ups.

## Output

- Review outcome.
- Findings.
- Residual risks.

## Gate

Review is approved, or work routes to `fix`, `diagnose`, `clarify`, `redesign`, or `split`.

## Outcome

`pass`, `fix`, `diagnose`, `clarify`, `redesign`, or `split`.

## Artifacts

- `review.md`
- PR checklist.
- Findings.

## Adapter

Specialized review agents can provide optional review modules when work-unit contracts are clear.
