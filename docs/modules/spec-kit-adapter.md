# Spec Kit Adapter Module

## Purpose

Map Spec Kit-style specification phases into SWECircuit modules.

## Input

- Product idea.
- Requirements.
- Existing spec package.

## Action

Support clarify, specify, plan, tasks, analyze, checklist, implement, and converge phases through SWECircuit artifacts.

## Output

- Spec.
- Plan.
- Tasks.
- Checklists.
- Implementation readiness.

## Gate

Spec-driven artifacts are clear and testable, or work routes to `clarify` or `split`.

## Outcome

`pass`, `clarify`, `split`, or `redesign`.

## Artifacts

- Feature package.
- Practice register.
- Adapter evaluation if installation is proposed.

## Adapter

GitHub Spec Kit remains optional. Use file-based SWECircuit contracts first.
