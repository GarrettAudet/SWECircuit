# Plan Module

## Purpose

Turn a spec into an implementation approach and verifiable task sequence.

## Input

- Spec.
- Architecture context.
- Verification requirements.

## Action

Define impacted areas, approach, risks, rollback, task breakdown, dependencies, and verification mapping.

## Output

- Implementation plan.
- Task list.
- Verification map.

## Gate

Tasks are small, sequenced, and verifiable, or work routes to `redesign`, `split`, or `clarify`.

## Outcome

`pass`, `redesign`, `split`, or `clarify`.

## Artifacts

- `plan.md`
- `tasks.md`
- `test-plan.md`

## Adapter

Spec Kit and Superpowers are optional adapter candidates for plan automation or guided planning.
